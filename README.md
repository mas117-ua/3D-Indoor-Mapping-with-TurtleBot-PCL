## 📝 About The Project

This project constitutes the **Practice 1: 3D Indoor Mapping and Line Following** of the VAR course. It focuses on developing a SLAM-like (Simultaneous Localization and Mapping) pipeline using **ROS 2**, **Gazebo**, and the **Point Cloud Library (PCL)**. 

The main objective is to control a simulated TurtleBot3 inside an environment ("Grandma Annie's House" from the RoCKIn competition) and build an incremental 3D map by capturing and registering point clouds generated by the onboard Kinect camera.

### Features
- Real-time 3D reconstruction from ROS `sensor_msgs::msg::PointCloud2`.
- Interactive algorithm selection at nodes start.
- Multi-step robust algorithmic pipeline.
- Continuous visualization via `pcl::visualization::CloudViewer`.

## ⚙️ The Algorithm Pipeline

The core logic is implemented in `node.cpp` (`get_pointclouds` package). It follows a rigorous mathematical and computational approach to align incoming point clouds incrementally:

### 1. Pre-Processing 🧹
Filtering NaNs out of the initial sweeps and applying a **VoxelGrid Downsampling** (leaf size of 2cm). This drastically reduces the 300,000+ points per frame into a manageable load while preserving the geometry.

### 2. Normal Estimation 📐
Normals are estimated (8cm radius) to understand surface orientations. Then, points with non-valid normals are discarded.

### 3. Keypoint Extraction 🔍
We implemented two methods you can choose from dynamically:
*   **ISS3D (Intrinsic Shape Signatures - Default):** Fast and efficient. Evaluates how much the geometry changes locally along its eigenvectors. Excellent for corners and edges.
*   **Harris3D:** Slower but precise. Mathematically evaluates the chaotic variance in normal vector directions within a radius to pinpoint the strongest structural corners.

### 4. Feature Description 🏷️
Each keypoint gets a numerical "Identity Card":
*   **FPFH (Fast Point Feature Histograms - Default):** Fast. Compares the keypoint only against its direct neighbors, returning a lightweight 33-bin histogram.
*   **PFH (Point Feature Histograms):** Heavy but highly precise. Forces cross-comparisons between *all* neighbors within the search radius. Returns a 125-value signature.

### 5. Matching & RANSAC Coarse Alignment 🧩
*   **Reciprocal Correspondences:** Ensures only mutually agreed matches between the current and previous frame are kept.
*   **RANSAC:** Random Sample Consensus finds the best transformation mathematically supported by the inliers, discarding false pairings (outliers).
*   **Kinematic Constraints:** The transformation collapses Height (Y) changes and non-Yaw rotations, respecting the TurtleBot planar movement behavior. *Sanity checks automatically ignore impossible physical speeds (jumps > 30cm or >15°).*

### 6. ICP Fine Refinement 🛠️
The **Iterative Closest Point** algorithm kicks in using the coarse guess from RANSAC. It iteratively pulls and rotates the entire cloud millimeter by millimeter until both clouds fit tightly together, ensuring zero seams in the global map.

## 🚀 Getting Started

### Prerequisites
*   Ubuntu 22.04 + ROS 2 Humble (or compatible distro)
*   Point Cloud Library (PCL) development tools
*   Gazebo Ignition / Classic depending on your setup.

### Installation

1. Clone the repo and enter workspace:
   ```bash
   git clone <YOUR_GITHUB_REPO_URL>
   cd VAR2026/ros2_ws
   ```
2. Build the package via colcon:
   ```bash
   colcon build --packages-select get_pointclouds
   ```
3. Source the environment:
   ```bash
   source install/setup.bash
   ```

### Execution

1. Make sure your Turtlebot Gazebo simulation is up and running in a separate terminal.
2. Launch the node:
   ```bash
   ros2 run get_pointclouds <executable_name>
   ```
3. An interactive menu will prompt you in the terminal to select your combinations of **Keypoints (ISS3D / Harris3D)** and **Descriptors (FPFH / PFH)**. 
4. Move your robot around using the teleop keys and watch the map incrementally grow in your separated viewer!
